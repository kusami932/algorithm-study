# Mistake Log

## 문제 정보

- 날짜:
- 사이트 / 문제 번호:
- 문제 이름:
- 언어:
- 결과:
- 독립 풀이 시간:
- 힌트 사용 여부:

## 1. 문제 요약

입력으로 무엇이 주어지고 무엇을 출력해야 하는지
2~4문장으로 작성한다.

## 2. 처음 떠올린 접근

처음에 어떤 방식으로 풀려고 했는지 작성한다.

## 3. 실제로 틀린 코드 또는 사고

문제가 된 코드나 논리를 작성한다.

## 4. 틀린 이유

단순히 "실수"라고 쓰지 말고,
왜 그 실수를 했는지 설명한다.

## 5. 실수 유형

- [ ] 문제 이해 오류
- [ ] 알고리즘 선택 오류
- [ ] 자료형 오류
- [ ] 초기화 오류
- [ ] 인덱스 오류
- [ ] 경계 조건 누락
- [ ] 시간복잡도 오류
- [ ] 입력 처리 오류
- [ ] 출력 형식 오류
- [ ] 언어 문법 오류
- [ ] 반례 검증 부족
- [ ] 기타:

## 6. 반례

내 코드가 실패하는 가장 작은 입력을 작성한다.

## 7. 올바른 핵심 원리

정답 풀이를 코드 없이 설명한다.

## 8. 다음에 잡아야 할 신호

비슷한 문제를 만났을 때
무엇을 보고 이 풀이를 떠올려야 하는지 작성한다.

## 9. 재발 방지 규칙

다음부터 실제로 수행할 행동을 작성한다.
